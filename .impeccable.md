# Design Context — mktr-platform (web admin)

The canonical Design Context lives at `../.impeccable.md` (lyfe-master root). Read that first — everything below is mktr-platform-specific overlay.

## Platform-Specific Notes

- **React 19 + Vite + shadcn/Radix + Tailwind CSS** (v3). Tailwind consumes the shared Tropic preset at `src/design/tailwind-preset.cjs`.
- **Theme wiring**:
    - `src/index.css` imports `./design/tropic.css` (generated HSL triplets for every shadcn semantic variable).
    - `tailwind.config.js` uses `presets: [require('./src/design/tailwind-preset.cjs')]`. No hand-written color/fontFamily extension lives in the config anymore.
    - Shadcn primitives (`src/components/ui/*`) resolve against standard semantic names (`bg-background`, `border-border`, `bg-primary`, etc.). The values are now Tropic instead of the default cool-blue.
- **Fonts**: Fraunces / Albert Sans / JetBrains Mono loaded via Google Fonts `@import` in `index.css`. `font-sans` → Albert Sans, `font-serif` → Fraunces, `font-mono` → JetBrains Mono.
- **Tropic-specific utility classes** available in addition to shadcn semantics: `bg-cream`, `bg-paper`, `text-terracotta`, `bg-sage`, `bg-butter`, `bg-plum`, `text-ink`, plus the `terracotta` scale (`terracotta-light`, `terracotta-dark`, `terracotta-muted`).

## Web Adapter Role

This app is an **adopter** of Tropic, not its source of truth. Do not:
- Hand-edit `src/design/*` — it is overwritten on `npm run sync:design`.
- Add palette hex values anywhere in `src/components/**`. Always use Tailwind semantic classes or the Tropic utility classes.
- Reintroduce `fontFamily: ['Inter']` or slate/indigo/cyan Tailwind colors. These are the pre-Tropic AI-default look; call `/impeccable:audit` if you suspect drift.

## Context Overlay

Mktr-platform is used on **desktop / tablet**, during daytime office hours, by a **small admin team** (one primary admin, occasional collaborator). Screens are data-dense: prospects tables, fleet maps, commission ledgers. Design decisions should favor:

- **Information density** over mobile-style spacious cards. Use tight `py-2` table rows, compact avatars, sidebar-first navigation.
- **Keyboard affordances** over mobile touch. Cmd-K command palette, keyboard shortcuts on listings, focus rings that are clearly visible.
- **Tropic warmth on a dense surface.** The warm palette is especially important on data-heavy screens — without it the admin falls back into the cool-blue spreadsheet-SaaS aesthetic.
